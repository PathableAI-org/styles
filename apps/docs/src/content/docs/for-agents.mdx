title: For Agents
description: Rules and conventions for AI coding agents working with PathableAI styles.
---

These rules govern how AI coding agents should work with PathableAI styles.

## Token usage

- **MUST** use design tokens from `@pathable/styles` rather than hard-coded values.
- **Prefer** semantic tokens (`--pathable-color-text`, `--pathable-color-surface`) over raw brand tokens for UI elements.
- **MAY** use USWDS utility classes (e.g., `bg-primary`, `text-white`, `padding-2`) for quick prototyping or HTML demos.

## Color

- **MUST NOT** introduce new brand colors without review. Use existing brand and semantic tokens.
- **MUST** check contrast ratios when combining foreground and background tokens.
- Use `--pathable-color-accent` for interactive/highlighted elements.
- When writing SCSS, **MUST** use the `uswds.color("token-name")` function to reference USWDS system tokens rather than `$theme-color-primary` directly.

## Typography

- **MUST** use `--pathable-font-heading` (Fredoka) for all heading elements.
- **MUST** use `--pathable-font-body` (Nunito) for all body text.
- **MUST NOT** introduce new font families.

## Spacing and layout

- **SHOULD** use the `--pathable-spacing-*` scale for margins and padding.
- **SHOULD** use `--pathable-radius-*` tokens for border-radius instead of arbitrary values.
- **MAY** use USWDS spacing utilities (`padding-*`, `margin-*`, `gap-*`) as a shorthand alternative to `--pathable-spacing-*` tokens.

## USWDS configuration

- ALL USWDS theme color overrides **MUST** be scoped exclusively within `_uswds-theme.scss`.
- Do **NOT** add `$theme-color-*` overrides to any other file.
- When upgrading USWDS: bump the version in `package.json`, rebuild, and verify no unexpected color changes.
